# OTD 區域計算器 v2 - 使用說明

## 🎯 快速開始

### 方法一：自動模式（推薦）⭐

1. **啟動程式**
   ```bash
   python otd_area_calculator_v2.py
   ```

2. **確認設定**
   - ✅ 確保「自動偵測 osu! 並開始錄製」已勾選（預設）
   - 此時 F10/F11 快捷鍵已停用，不會與 osu! 衝突

3. **開始遊玩**
   - 直接開啟 osu!
   - 程式會自動偵測並開始錄製
   - 狀態會顯示：🎮 偵測到 osu! - 自動開始錄製...

4. **遊玩 1-2 首歌**
   - 正常遊玩即可
   - 程式會即時顯示已記錄的數據點數量

5. **結束遊玩**
   - 關閉 osu! 或切換到其他視窗
   - 程式會自動停止錄製

6. **查看結果**
   - 點擊「計算區域」
   - 將結果套用到 OpenTabletDriver

---

### 方法二：手動模式

1. **取消勾選「自動偵測 osu!」**
   - F10/F11 快捷鍵會自動啟用
   - 手動控制按鈕會變為可用

2. **按 F10 或點擊「開始錄製」**

3. **遊玩 osu!**

4. **按 F11 或點擊「停止錄製」**

5. **點擊「計算區域」**

---

## 📋 功能說明

### 自動偵測 osu!
- 每秒檢查 osu! 是否正在執行
- 支援 `osu!.exe` 和 `osu!lazer.exe`
- 自動開始/停止錄製
- **自動停用 F10/F11 快捷鍵**，避免與 osu! 衝突

### 手寫板自動偵測
- 自動識別 Wacom CTL-4100
- 顯示手寫板規格（最大座標、實體尺寸）
- 支援其他 Wacom 型號

### 智能計算
- 自動過濾極端值（2% 離散點）
- 自動比例校正（確保畫圓是圓）
- 精確的座標轉換（counts → mm）

## ❓ 常見問題

**Q: 為什麼自動模式下按鈕是灰色的？**  
A: 這是正常的！自動模式下不需要手動控制，所以按鈕和 F10/F11 都會停用。

**Q: 如何切換回手動模式？**  
A: 取消勾選「自動偵測 osu!」即可，按鈕和快捷鍵會立即啟用。

**Q: 程式沒有偵測到 osu!？**  
A: 確認 osu! 的執行檔名稱是 `osu!.exe` 或 `osu!lazer.exe`。

**Q: 可以在 osu! 開啟時啟動程式嗎？**  
A: 可以！程式會立即偵測到 osu! 並開始錄製。

**Q: 需要多少數據點？**  
A: 建議至少 10000 個點，通常遊玩 1-2 首歌就足夠了。

## 🎮 使用技巧

1. **選擇代表性的圖譜**
   - 選擇您常玩的難度
   - 包含跳圖和串流的混合圖譜最佳

2. **正常遊玩即可**
   - 不需要刻意測試範圍
   - 程式會自動過濾極端動作

3. **多次測試**
   - 可以重複錄製並比較結果
   - 不同遊玩風格可能有不同結果

4. **微調結果**
   - 如果覺得太小，可以手動放大 10-20%
   - 如果覺得太大，可以縮小 10-20%
